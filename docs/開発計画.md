# 開発計画

## 1. 目的
- `Laravel API + React SPA` で社内教育システムを構築する。
- 受講者・レビュー・管理者の3ロール運用を可能にする。
- 提出、レビュー承認、進捗集計、教材管理、自動判定課題を一体で提供する。

## 2. スコープ
- 招待制認証、マイページ、コース閲覧、提出、レビュー。
- 管理機能（コース/レッスン/ユーザー/ラベル/招待）。
- PDF配信（MinIO）、判定実行（Docker隔離）。
- APIテスト、フロントテスト、E2E、CI。

## 3. 実装構成
- `backend/`: Laravel 12 API
- `frontend/`: React + TypeScript + Vite
- `infra/`: Docker Compose / Dockerfiles / MinIO初期化
- `docs/`: 要件・設計・運用文書

## 4. 実施順序
1. モノレポ雛形作成と依存導入。
2. DBスキーマ・モデル・API・認可・判定基盤実装。
3. React画面実装（受講者/レビュー/管理者）。
4. テスト・CI整備。
5. 文書整備と受け入れ観点の最終確認。

## 5. 品質方針
- バックエンド: `php artisan test` を常時通す。
- フロント: `npm run build`, `npm run test`, `npm run test:e2e` を通す。
- 主要フロー（提出→レビュー→合格）をFeatureテストで担保する。
