# テスト計画書

## 1. Backend (Laravel)
- `php artisan test`
- Featureテスト観点:
  - 招待受諾とユーザー作成
  - 無効ユーザーのログイン拒否
  - 提出→レビュー→合格フロー
  - 受講者の管理API拒否

## 2. Frontend (React)
- `npm run test`
- Vitestで画面レンダリングの基本検証。

## 3. E2E
- `npm run test:e2e`
- Playwrightでログイン画面のスモークテスト。

## 4. Build検証
- `npm run build`
- TypeScriptビルドとVite本番ビルドが成功すること。

## 5. 回帰確認（手動）
- 受講者で提出できること。
- レビュー担当がコメント必須で承認/差戻しできること。
- 管理者がコース/レッスン/ユーザー/ラベルを操作できること。
